## 0. 공통 사항 (Common)

### 0.1 사전 조건 (Common Preconditions)
- 테스트 계정이 KMS 서비스에 정상적으로 등록되어 있음
- 테스트용 프로젝트/리전이 사전 세팅되어 있음
- 충분한 권한 (KMS Admin, KMS User, Viewer)을 가진 계정이 각각 준비됨
- 네트워크/세션 상태 정상

### 0.2 공통 UI/UX 검증 포인트
- [ ] 페이지 로드 시 1차 콘텐츠가 2초 이내 표시되는지
- [ ] 라이트 / 다크 테마 토글 시 모든 요소가 누락 없이 전환되는지
- [ ] 한국어 UI에 잘림(truncation) 또는 줄바꿈 깨짐 현상이 없는지
- [ ] 키보드 포커스 이동(Tab) 시 시각적 포커스 링이 명확히 보이는지
- [ ] 모든 버튼/링크에 hover, active, disabled 상태가 정의되어 있는지
- [ ] 오류·성공 토스트 메시지가 일관된 위치(우상단 또는 상단)에 노출되는지

### 0.3 용어
| 용어 | 정의 |
| --- | --- |
| 고객 관리형 키 (CMK) | 고객이 직접 생성·관리하는 암호화 키 |
| 플랫폼 관리형 키 (PMK) | 플랫폼이 자동으로 발급·운용하는 키 |

| 서비스 해지 | 키와 연결된 KMS 서비스 자체 종료 (즉시 / 예약 / 취소) |

#### TC-CMK-TERM-004 — 해지 가능 권한 검증
| 단계 | 1. Viewer/일반 사용자 계정으로 해지 시도 |
| --- | --- |

## 5. UI/UX 시각 검증 체크리스트

### 5.1 레이아웃
- [ ] 모든 페이지 사이드바·헤더 고정 동작 일관
- [ ] 컨텐츠 영역 최소/최대 폭(1280–1600px) 가독성
- [ ] 카드·테이블의 패딩·여백이 디자인 토큰과 일치

### 5.2 색상·타이포
- [ ] Primary / Danger / Success / Warning 컬러 사용처 일관
- [ ] 본문 14px / 보조 12px / 제목 18~24px 위계
- [ ] 다크 모드의 배경 대비(WCAG AA) 만족

### 5.3 상태 표시
- [ ] 활성 / 비활성 / 해지됨 / 해지 예약 / 삭제 진행 중 배지 색상·아이콘 구분
- [ ] 로딩 스켈레톤이 ≥300ms 작업 시 노출
- [ ] 빈 상태 / 에러 상태에 일관된 일러스트 사용

### 5.4 인터랙션
- [ ] 모든 모달은 ESC 키 + 외부 영역 클릭으로 닫힘 (단, 위험 액션 모달은 외부 클릭으로 닫히지 않음)
- [ ] 토스트는 4초 후 자동 사라짐 + 수동 닫기 가능
- [ ] 폼 제출 중에는 버튼이 로딩 상태(spinner)로 전환되고 중복 클릭 방지

### 5.5 접근성
- [ ] 모든 인터랙티브 요소 Tab 순서 자연스러움
- [ ] 스크린리더용 aria-label 설정
- [ ] 색상만으로 상태를 전달하지 않음 (텍스트/아이콘 병행)

> Figma MCP 호출 한도 도달로 모든 프레임을 세부 분석하지는 못했습니다. 아래 항목은 디자인 원본을 추가 확인하여 시나리오를 보강할 필요가 있습니다.
